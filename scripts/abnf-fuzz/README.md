# ABNF fuzz suite

Property-style fuzz harness that drives each per-header parser with inputs
generated by [`abnfgen`](https://www.quut.com/abnfgen/) from RFC 3261
grammars. Asserts the parser never rejects a grammar-valid input and never
silently misparses one.

The grammars under [grammars/](grammars/) are extracted verbatim from
[xsnpdngv/SIP-ABNF](https://github.com/xsnpdngv/SIP-ABNF) (which itself is
an aggregate of RFC 3261 §25.1 plus extension RFCs). Some productions are
tightened where the pure ABNF is ambiguous with the parser's RFC 3261
§20.10 angle-bracket policy (see comments inside each grammar file).

## One-time setup

```bash
curl -sSL https://www.quut.com/abnfgen/abnfgen-0.20.tar.gz | tar xz -C /tmp
(cd /tmp/abnfgen-0.20 && ./configure && make)
export ABNFGEN=/tmp/abnfgen-0.20/abnfgen
```

## Run

```bash
N=5000 scripts/abnf-fuzz/run-suite.sh
```

Output: one line per target with total / accepted / policy-reject / buggy-reject
/ silent-misparse counts. Full per-target reports (with rejected samples) land
in `${OUT:-/tmp/abnf-fuzz-out}/<target>.report.json`.

## Targets

| Target          | Start symbol         | Parser exercised |
|-----------------|----------------------|------------------|
| `sip-uri`       | `SIP-URI`            | `validateStrictSipUri`, `parseSipUriString` |
| `from`          | `from-spec`          | `parseNameAddr` + strict URI |
| `pai`           | `pai-header-value`   | `splitTopLevelCommas` + `parseNameAddr` per entry (sip/sips/tel) |
| `contact`       | `contact-value`      | `parseContact` + strict URI |
| `via`           | `via-header-value`   | `parseVia` |
| `cseq`          | `cseq-value`         | `parseCSeq` |
| `rack`          | `rack-value`         | `parseRack` |
| `replaces`      | `replaces-value`     | `parseReplaces` |
| `refer-to`      | `refer-to-value`     | `parseReferTo` + `findUriEmbeddedHeadersStart` |
| `request-line`  | `request-line`       | Request-URI extraction + strict URI |

## Interpreting results

The driver classifies each rejection:

- **buggy**: a rejection that does not match any known semantic-policy
  pattern. A non-zero `buggy` count usually means a real parser bug
  (over-narrow stop set, wrong index pivot, etc.).
- **policy**: rejection reason matches a documented semantic constraint
  beyond pure ABNF — `port out of range`, `port == 0`, `empty hostport`,
  multiple `@` in userinfo, etc. The grammar is intentionally a bit looser
  than the spec on these, so a few percent of samples land here.
- **silentMisparse**: parser accepted the input, but the parsed structure
  is obviously wrong (e.g. URI literally contained `@` but `parsed.user`
  came back undefined).

A clean run has `buggy = 0` and `silentMisparse = 0` across all targets.
